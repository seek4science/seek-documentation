This is an outline of plans the core FAIRDOM-SEEK development team will be working on in the near future.

FAIRDOM-SEEK is developed following an Agile development process, meaning priorities may change - however, these items are currently high priorities.

They are expected to be added to FAIRDOM-SEEK in roughly the order they are presented here.

## Now

- Ability to merge accounts
- UI improvements to the admin area
- Enhanced ISA-JSON export (via DataHub) supporting brokering tools to submit to repositories
- Harmonize and tidy up the submission forms, and remove the DataFile upload wizard to improve consistency
- Visible history log for Sample changes
- Improving search to support more advanced queries

## Next

- Redis / Sidekick
- Rails 8.1
- Checking the content security policies (inline JS/CSS, nonces)

## Later

- S3 support
- RO-Crate for ISA/Collections

Please also see the [WorkflowHub Roadmap](https://about.workflowhub.eu/roadmap/),
which is being built on and in conjunction with FAIRDOM-SEEK.
